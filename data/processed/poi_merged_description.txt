/*
  통합 POI 데이터 (raw + AI analysis merged)

  본 파일은 관광 API 원본 데이터(raw_poi)와
  AI 분석 결과(poi_analysis)를 통합한 POI 마스터 데이터셋입니다.

  목적
  - 추천 시스템 / POI 임베딩 / 개인화 추천 / 일정 생성 알고리즘에서
    한 번에 접근 가능한 통합 POI 정보를 제공
  - 원본 텍스트(overview) + AI 구조화 필드(themes, mood, visitor_type…)를
    모두 포함하여 피처 엔지니어링 및 알고리즘 개발을 용이하게 함

  구성
  ┌ raw 데이터 (수집한 원본)
  │  poi_id, type, contenttypeid, name, gu_name, location, category
  │  overview, media, intro, naver, naver_match
  │
  └ AI 분석 결과 데이터 (LLM 기반)
     poi_type, themes, mood, visitor_type, best_time, best_time_flags
     duration, activity, photospot, indoor_outdoor
     keywords, summary_one_sentence, avoid_for, ideal_schedule_position

  활용 위치
  - 모델 임베딩 생성 (예: bge-m3 텍스트 입력 구성)
  - 추천 시스템 feature 엔지니어링
  - 일정 생성 알고리즘 (시간대/체류시간/동선/피로도 등)
  - POI 필터링 및 UI 요소 구성

  ⚠️ 주의
  - overview는 원본 텍스트 → 길고 중복/광고성 표현 포함 가능
  - AI 분석 결과는 요약/정제된 2차 정보 → 완전히 정확하지 않을 수 있음
  - naver 필드는 null 가능 → 인기도/평점 피처 생성 시 결측 처리 필수
  - raw와 AI 결과가 서로 충돌할 수 있으므로 정제 로직 필요할 수 있음

  향후 확장 가능성
  - review_summary, blog_summary, category_embedding 등 추가 예정
  - multilingual 필드 확장 가능 (en/jp/cn)
  - poi_id 기준 외부 DB 조인 가능

  이 데이터는 서비스 응답 결과가 아니라,
  추천/모델/인사이트 개발을 위한 "백엔드·AI 내부용 마스터 데이터"임.
*/
